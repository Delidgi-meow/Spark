# Spark Dating App — SillyTavern Extension v3.0.0

Плавающая кнопка-телефон с приложением знакомств. Свайпы, чаты, матчи — всё внутри. Основной чат ST используется для свиданий и реакций персонажа.

## Установка
1. Папка лежит в `public/scripts/extensions/third-party/Spark/`.
2. F5 → **Extensions → Manage extensions** → включи **Spark Dating App**.
3. Справа снизу появится телефончик ✦.

## v3 что нового
- **Persistence через `chat_metadata`** — переписки и матчи переживают рестарт ST.
- **Per-chat хранение** — в каждом чате ST свой Spark-стейт.
- **Синк с основным ботом** — бот знает что ты делаешь в Spark, может ссылаться (вкл/выкл в настройках).
- **Strip `<think>`** — теги мышления вырезаются из ответов парней.
- **Connection profile** — можно указать отдельный профиль API для Spark (без reasoning-моделей).
- **Загрузка картинок в чат** — скрепка → файл → отправляется в переписку, парень реагирует.
- **Аватары** — загрузить файл ИЛИ сгенерировать через Stable Diffusion расширение (`/sd`).
- **Ростер из лорбука** — описания парней можно вынести в world-info, расширение их подгрузит.
- **Settings tab** (шестерёнка) — все настройки в самом приложении.

## Настройка ростера через лорбук
1. Создай лорбук в ST (например `spark-roster-data`).
2. Добавь запись с **comment** = `spark-roster` и **content** = JSON (см. `LOREBOOK_TEMPLATE.js`).
3. В настройках Spark впиши имя лорбука → «Перезагрузить ростер».
4. Если поле пустое — используются встроенные дефолтные карточки.

## Connection profile
Если основной чат идёт на reasoning-модели (DeepSeek-R1, o1) которые лепят `<think>`, создай отдельный Connection Profile в ST для обычной модели и впиши его имя в настройках Spark. Расширение переключится на него только для генерации ответов парней.

## Консоль
```js
sparkOpen()           // открыть приложение
sparkDebug()          // состояние текущего чата
sparkReset()          // сбросить состояние
sparkReloadRoster()   // перезагрузить ростер из лорбука
sparkRoster()         // текущие карточки
```

## Файлы
- `manifest.json` — метаданные
- `index.js` — инициализация, FAB, события
- `ui.js` — рендер всех экранов (свайп / матчи / чат / настройки)
- `engine.js` — вызовы LLM, strip think, инжект в основной чат
- `scoring.js` — матчинг
- `roster.js` — дефолтные карточки + загрузчик из лорбука
- `state.js` — `chat_metadata` (per-chat) + `extension_settings` (глобальное)
- `style.css` — стили
- `LOREBOOK_TEMPLATE.js` — справка по формату лорбука
